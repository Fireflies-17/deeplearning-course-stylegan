# 课程设计梳理

更新时间：2026-06-06

## 1. 课题与目标

选题：

> **基于 StyleGAN 的图像生成**

暂定报告题目：

> **基于 StyleGAN 的图像生成实现与实验分析**

研究目标：

1. 建立可复现的 StyleGAN2-ADA 训练、生成和评价流程；
2. 在一个正式数据集上完成可信的基线复现；
3. 根据初步实验发现问题，再选择一个方向进行公平对比和深入分析；
4. 形成由实验结果支持的结论，而不是预设改进一定有效。

当前不提前锁定最终研究问题。优先关注有限数据条件下的生成质量、训练稳定性和判别器
过拟合问题。

## 2. 硬性要求

- 必须提交实验代码和中文课题报告；
- 报告采用小论文形式，至少 20 页；
- 报告应包含摘要、引言、研究方法、实验与结果分析、结论和参考文献；
- 代码需要可复现，说明环境、数据、运行方法和实验结果；
- 截止时间：**2026-06-30 24:00**；
- 提交渠道：智慧树；
- 独立完成，严禁抄袭。

争取高分需要完成可靠对比、参数分析或消融实验，并给出自己的分析，不能只展示单次复现
结果。

## 3. 当前技术选择

| 项目 | 当前选择 |
|---|---|
| 统一实验后端 | NVIDIA StyleGAN2-ADA PyTorch |
| 固定后端提交 | `d72cc7d041b42ec8e806021a205ed9349f87c6a4` |
| 目标 GPU | RTX PRO 6000，4 卡机；每组训练用 2 卡（方案 C：两组并行各占 2 卡） |
| 目标镜像 | Ubuntu 22.04、Python 3.12、PyTorch 2.8、CUDA 12.8 devel |
| 关键依赖口径 | `torch==2.8.0+cu128` 按 NumPy 2.x 编译，必须 `numpy>=2.0,<2.3`（NumPy 1.x 会崩 `torch.from_numpy`，详见 `environment.yml`） |
| 当前阶段 | P2 因素矩阵实验（P0/P1 已完成，基线 FID50k_full≈13） |
| 正式数据集 | LSUN Church Outdoor，256x256，100,000 张训练图像子集 |
| 主研究方向 | 有限数据下的 ADA 行为、数据规模与 ADA target 诊断（见 `report/p2_experiment_plan.md`） |

选择 StyleGAN2-ADA 的原因：

- 可以在同一框架中完成从头训练、迁移学习、ADA 和 FreezeD 等对比；
- 适合有限数据场景，实验组容易控制；
- 官方代码包含训练、生成和正式评价入口；
- 相比同时整合多个不同代码库，更容易保证公平性和可复现性。

## 4. 实验路线

| 阶段 | 主要工作 | 完成标志 |
|---|---|---|
| P0：流程验证 | 验证环境、数据、训练、快照加载、生成和指标入口 | 目标 GPU 上完整短流程通过 |
| P1：基础复现 | 选择正式数据集并完成 StyleGAN2-ADA 基线训练 | 获得可信样本、曲线和指标 |
| P2：问题诊断 | 分析数据规模、训练预算、过拟合和失败模式 | 找到稳定且可解释的问题 |
| P3：方向选择 | 根据诊断结果确定最终研究问题和实验组 | 锁定正式实验方案 |
| P4：正式实验 | 完成对比、消融、参数分析和必要重复 | 形成支持报告结论的证据 |

P1 基线（100k、ADA、target=0.6、2000 kimg）已完成，FID50k_full≈13。在此基础上，原先设想的
L0–L3 探索性分组已细化、落地为正式的因素矩阵 E1–E5（统一 1500 kimg 公平预算、`mirror=false`
离线评估），详见 `report/p2_experiment_plan.md`：

| 实验 | 数据规模 | 增强方式 | 对应原 L 组 | 作用 |
|---|---:|---|---|---|
| E1 | 100k | ADA target=0.6 | L0 | 中心基线（取 1512 kimg 快照；2000 kimg 用于最终展示） |
| E2 | 100k | 无增强 | L1 | 分析 ADA 的作用 |
| E3 | 100k | 固定增强 p=0.2 | — | 无增强—固定—自适应增强完整对照 |
| E4 | 50k | ADA target=0.6 | L2 | 数据规模与过拟合 |
| E5 | 100k | ADA target=0.4 | L3 的参数侧 | ADA target 敏感性 |

说明：由于该矩阵每组都是单一变量受控对照，已**同时承担原 P2 诊断与 P4 正式对比**的职能，
P2/P3/P4 的阶段划分实际合并为"围绕基线的一因素矩阵 + 多类非训练分析"。训练预算本身不再
单列为研究因素，仅用 2000 kimg 模型展示更充分训练后的效果。

迁移学习和 FreezeD 不纳入本主线。只有当该矩阵无法形成可解释问题，才考虑切换到 FFHQ、AFHQ
或 StyleGAN2 与 StyleGAN3 的等变性比较。

## 5. 数据集选择

正式数据集：

- LSUN Church Outdoor 训练集；
- 固定使用 100,000 张训练图像子集；
- 256x256 RGB；
- 无类别标签，P1 使用非条件训练；
- 转换文件固定为 `data/processed/lsun-church-256-100k.zip`。

AFHQ Cat、FFHQ、ImageNet 和 LSUN Bedroom 暂时降为备选方向，不进入 P1。主要原因不是方法不适合，
而是 AFHQ/FFHQ 有外网下载和人脸伦理压力，ImageNet 类别过杂且生成难度过高，LSUN Bedroom 全量
数据搬运成本更重。LSUN Church 在视觉效果、业界认可度、数据规模和可操作性之间更均衡。

正式实验必须固定数据源、校验值、预处理方式和分辨率，并记录下载日期、文件大小、
转换命令和随机种子。

## 6. 实验规范

公平性控制：

- 主实验每次只改变一个指定因素；
- 使用相同数据、预处理、训练预算和评价样本数；
- 训练预算使用 `kimg`，不使用 epoch 直接比较；
- 保持全局 batch size 一致或明确记录差异；
- 探索性短跑可使用单个随机种子，关键结论尽量使用多个随机种子验证；
- 所有参数选择和实验组筛选规则都要记录。

主要评价内容：

| 类别 | 指标或输出 |
|---|---|
| 生成质量 | FID、KID、Precision |
| 生成多样性 | Recall、随机样本、潜空间插值、风格混合 |
| 训练稳定性 | 生成器与判别器损失、ADA 概率、失败现象 |
| 资源效率 | 训练速度、GPU 小时、峰值显存 |
| 训练集记忆 | 生成样本与训练集最近邻对比 |

FID 是主要指标，但最终结论不能只依赖单个数值，必须结合曲线、样本和失败案例分析。
